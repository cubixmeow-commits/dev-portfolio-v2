# Cookbooks for previously empty categories

Five executable Cookbooks filling every remaining empty primary category. Each follows the public-knowledge → SousMeow conversion pattern used for Study Plan and Clear Email.

## Complexity Gate (Product Law 002)

All five passed on paper before authoring: audience fit, known-facts pantry only, invent-nothing prompts, judgment inside Recipes and checks, surface kept to one job per Cookbook.

## Batch overview

| Category | Cookbook | Source inspiration (ideas only) | Shape |
| --- | --- | --- | --- |
| Software & Product | Write a Feature Spec | Public agile user-story / INVEST teaching; lightweight PRD patterns | 3 stages / 6 recipes / ~50 min |
| Design & Brand | Name Your Brand Voice | Public brand-voice teaching; plain-language brand guidance | 3 stages / 7 recipes / ~30 min |
| Research & Insights | Compare Three Competitors | University entrepreneurship competitive-analysis teaching | 3 stages / 8 recipes / ~55 min |
| Planning & Productivity | Make a Criteria Decision | University advising structured decision worksheets | 2 stages / 2 recipes / ~20 min |
| Personal Projects | Finish a Personal Project | SMART-goal advising; simple work-breakdown teaching | 3 stages / 9 recipes / ~70 min |

No guideline or textbook wording was copied. Sample projects (KinCalendar Quiet Hours, Harbor Thread, BatchBowl, career options, Desk Garden) are fictional SousMeow content.

## Structural uniqueness

Recipe counts used: **6, 7, 8, 2, 9** (none shared with the prior library). Stage shapes: **2-2-2, 2-3-2, 2-4-2, 1-1, 3-3-3**.

## Reusable pattern

1. Empty category + everyday job.
2. Public method family (university / official / widely taught).
3. Ban source wording; invent a concrete sample project.
4. Pantry = facts the user already knows.
5. Teach retry advice inside `why_it_matters` and checks.
6. Keep recipe counts and stage shapes unique across the library.
