# Second Cookbook per category

Twelve executable Cookbooks so every primary category has exactly two. Builds on the empty-category batch and the Study Plan / Clear Email public-source pattern.

## Complexity Gate (Product Law 002)

Each Cookbook uses known-facts pantry only, invent-nothing prompts, and teaching inside `why_it_matters` / checks. Stage shapes are unique across the full twenty-four Cookbook library so Product Law can keep recipe counts moderate when a category already has a longer first Cookbook.
